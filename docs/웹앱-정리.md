# 이력서 맞춤화 웹앱 — 정리 문서

채용공고와 이력서를 입력하면 **분석 → 맞춤 이력서 생성 → 편집·PDF 출력 → 면접 예상 질문 추출**까지 한 흐름으로 처리하는 1인용 웹 도구.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 목적 | 마스터 이력서(경험 풀) + 채용공고로 기업 맞춤형 이력서를 빠르게 생성 |
| 사용자 | 본인 1인 (비밀번호 게이트로 보호) |
| 저장 | 서버 DB 없음 — 작업물은 **브라우저 localStorage**에 저장 |
| AI | OpenAI API 스트리밍(분석·생성·면접질문) |

---

## 2. 기술 스택

- **프레임워크**: Next.js 16 (App Router) + React 19
- **언어**: TypeScript
- **스타일**: Tailwind CSS 3 (+ `globals.css`의 디자인 시스템 토큰)
- **마크다운**: react-markdown + remark-gfm(표 렌더링)
- **PDF 텍스트 추출**: pdf-parse
- **AI**: openai SDK (모델 기본 `gpt-4o`)
- **테스트**: Vitest + Testing Library
- **ID 생성**: nanoid

실행 포트: **3030** (`npm run dev` / `npm run start`)

## 4. 화면 구성

### 게이트 (`/gate`)
- `APP_PASSWORD` 설정 시 활성. 접근 코드 입력 → 쿠키 발급 후 진입.

### 내 이력서 (`/master`)
- **마스터 / 일반 이력서를 탭으로 각각** 입력·수정하고 **자동 저장**(localStorage). 마지막 저장 시각 표시, PDF·이미지 업로드 지원.
- 새 분석(홈)에서 선택한 종류의 저장 이력서를 **자동으로 불러온다** → 매번 붙여넣을 필요 없음.

### 경험 브레인스토밍 (`/brainstorm`, `/brainstorm/[id]`)
- 막연한 기억을 **1~2개씩 질문으로 끌어내** 구체적 경험(문제·역할·성과)으로 정리하는 대화형 채팅.
- 다중 턴 대화는 `/api/brainstorm`(메시지 히스토리 → 스트리밍)으로 처리, **대화는 localStorage에 세션별 저장**.
- 어시스턴트 메시지마다 **[+ 마스터 이력서에 추가]** 버튼으로 STAR 정리 결과를 풀에 덧붙인다.

### 홈 (`/`)
- 좌측 사이드바(저장 문서 목록) + 본문.
- **이력서 입력**: 마스터/일반 라디오(설명 포함), 텍스트 붙여넣기 또는 PDF 업로드.
- **공고 입력**: 링크에서 가져오기 또는 텍스트 붙여넣기.
- **분석하기** → 스트리밍 결과 표시 → **작업 화면 열기**로 이동.
- ①②③ 온보딩 스텝, 분석 중 스켈레톤 표시.

### 프로젝트 화면 — 두 페이지로 구분
하나의 프로젝트(저장된 문서)는 사이드바와 상단 탭에서 **1. 분석 / 2. 이력서 작성**으로 나뉜다. 제목 인라인 편집·자동저장은 두 페이지 공통.

**1. 분석 페이지 (`/resume/[id]`)**
- 저장된 분석 결과 표시 + 채용공고 원문 펼쳐보기.
- (마스터) **이력서 작성하러 가기 →** 버튼.

**2. 이력서 작성 페이지 (`/resume/[id]/write`)**
- **맞춤 이력서 생성/재생성**(마스터 문서만, 재생성 시 확인 모달), **맞춤 이력서 검토**, **PDF 출력**.
- 마크다운 **편집·미리보기 2분할** 에디터.
- **맞춤 이력서 검토**: 생성된(선택된) 이력서를 공고와 비교해 1차 심사관 시선으로 진단(검토자 시뮬레이션, `/api/review`). ※1.분석은 마스터 풀↔공고(선별 가이드용), 검토는 선택본↔공고로 역할이 다름.

**3. 면접 질문 페이지 (`/resume/[id]/interview`)**
- 완성된(또는 입력) 이력서로 인성/기술 예상 질문을 추출(`/api/interview`). 재추출 가능.

## 5. 핵심 사용자 흐름

```
[게이트] → [홈: 이력서+공고 입력] → [분석] → [작업 화면 열기]
   → (마스터) 맞춤 이력서 생성 → 편집 → PDF 출력
   → 면접질문 추출
```

- **마스터 이력서**: 여러 경험을 모아둔 원본. 공고마다 맞춤 이력서 생성 가능.
- **일반 이력서**: 해당 공고 분석만 제공(맞춤 생성 없음).

## 6. API 엔드포인트

| 경로 | 역할 | 입력 | 출력 |
|------|------|------|------|
| `POST /api/analyze` | 공고 대비 이력서 분석 | resumeText, jobPosting | 분석 마크다운(스트림) |
| `POST /api/generate` | 맞춤 이력서 생성 | masterResume, jobPosting, analysis | 이력서 마크다운(스트림) |
| `POST /api/interview` | 면접 예상 질문 추출 | resumeText | 질문 마크다운(스트림) |
| `POST /api/extract-pdf` | PDF → 텍스트 | PDF 파일 | 텍스트 |
| `POST /api/fetch-job` | 공고 링크 → 본문 | url | 텍스트 |
| `POST /api/gate` | 비밀번호 검증 | pw | 인증 쿠키 |

모든 AI 호출은 **IP당 일일 호출 제한**(`DAILY_LIMIT`) 적용. 초과 시 `RATE_LIMITED`.

## 7. 데이터 모델 (localStorage)

```ts
type DocKind = "master" | "general";

type SavedDoc = {
  id: string;          // nanoid(8)
  title: string;
  kind: DocKind;
  createdAt: string;   // ISO
  jobPosting: string;  // 공고 원문
  analysis: string;    // 분석 결과 md
  resumeMd: string | null;     // 생성·편집된 이력서 md
  interviewMd: string | null;  // 면접 질문 md
  sourceResume: string;        // 입력 이력서 원문
};

type MasterResume = { text: string; updatedAt: string };
```

- 키: `resume-app:documents`, `resume-app:masterResume`
- **주의**: 데이터는 해당 브라우저에만 저장됨(다른 기기·브라우저에서 안 보임).

## 8. 환경 변수

| 변수 | 필수 | 기본값 | 설명 |
|------|------|--------|------|
| `OPENAI_API_KEY` | ✅ | — | OpenAI API 키 |
| `OPENAI_MODEL` | | `gpt-4o` | **기본(고품질) 티어** — 분석·이력서 생성에 사용 |
| `OPENAI_MODEL_ECONOMY` | | `gpt-4o-mini` | **절약 티어** — 면접질문에 사용 |
| `OPENAI_MODEL_ANALYZE` | | (절약 티어) | 분석 작업 모델 개별 지정 |
| `OPENAI_MODEL_GENERATE` | | (기본 티어) | 생성 작업 모델 개별 지정 |
| `OPENAI_MODEL_INTERVIEW` | | (절약 티어) | 면접질문 작업 모델 개별 지정 |
| `OPENAI_MAX_TOKENS` | | `16000` | 출력 토큰 상한 |
| `DAILY_LIMIT` | | `30` | IP당 일일 호출 한도 |
| `APP_PASSWORD` | | (빈값) | 설정 시 비밀번호 게이트 활성 |

### 작업별 모델 매핑

| 작업 | 기본 티어 | 비고 |
|------|----------|------|
| 공고 분석 (`analyze`) | **기본**(gpt-4o) | 분석 결과가 생성 입력 — 품질 우선 |
| 이력서 생성 (`generate`) | **기본**(gpt-4o) | 최종 산출물 — 품질 우선 |
| 면접질문 (`interview`) | 절약(gpt-4o-mini) | 출력량 큰 파생물, 비용 절감 |

우선순위: `OPENAI_MODEL_<TASK>` (개별 지정) > 작업별 티어 기본값. 면접질문도 기본 모델로 올리고 싶으면 `OPENAI_MODEL_INTERVIEW=gpt-4o`로 지정하면 된다.

## 10. 디자인 시스템

`app/globals.css`에 공통 클래스로 정의:

## 11. 개선 이력 (UX/UI)

1. **기능·예외처리**: 인라인 검증, 자동저장 상태 표시, 재생성 확인, 조건부 버튼 노출, 뒤로가기, 문서 삭제/생성일.
2. **디자인 시스템 도입**: 중립 3톤 컬러 + 카드 + 공통 input/button 클래스, 타이포 스케일 정리.
3. **인앱 토스트/모달**로 네이티브 팝업 대체.
4. **스트리밍 스켈레톤** + 홈 온보딩 스텝 추가.
5. **편집기 글자 겹침 버그 수정**: 고정 `70vh` 컨테이너 안에서 내용이 넘쳐 다음 섹션과 겹치던 문제 → 편집·미리보기 박스에 자체 고정 높이(`60vh`)+내부 스크롤 적용. 미리보기 `whitespace-pre-wrap` 제거 및 제목/목록 간격 직접 지정.
